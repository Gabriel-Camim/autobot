# Stack tecnico

Este documento consolida a stack técnica de Gabriel e também registra a stack atual do portfólio/Autobot. Ele não deve representar apenas uma arquitetura antiga do portfólio.

Quando a pergunta for genericamente sobre `stack`, a resposta deve priorizar a stack técnica geral de Gabriel e, em seguida, contextualizar a stack atual do portfólio/Autobot quando fizer sentido.

## Stack técnica de Gabriel

### Dados, IA e automações

- SQL, modelagem relacional, schemas, KPIs, projeções, forecast/reforecast, governança de dados, lineage, rastreabilidade e documentação de métricas.
- Power BI e Superset.
- n8n, APIs REST, webhooks, JSON, OpenAI API, Gamma API, parsing de respostas, tratamento de erros, pipelines de dados textuais, integrações entre ferramentas e workflows operacionais.

### Desenvolvimento e arquitetura

- JavaScript, Python, HTML/CSS e React/Vite.
- GitHub, Visual Studio Code, Vercel, Render e Docker.
- Backend, APIs internas, arquitetura modular, autenticação, permissões, CI/CD, banco multitenant e metodologias ágeis.
- PostgreSQL para persistência relacional quando o projeto exigir banco relacional.

## Stack atual do portfólio/Autobot

- Backend em Python e FastAPI.
- LangChain para fluxos de RAG e orquestração quando aplicável.
- PgVector como banco vetorial.
- PostgreSQL como banco relacional.
- OpenAI API com GPT-5.5 para respostas do agente.
- GPT Realtime-2 para conversação em tempo real por WebRTC.
- React com Vite no frontend.
- Deploy com Vercel no frontend e Render ou Railway no backend, quando aplicável.

## Observações de curadoria

- Não citar GPT-4o como modelo atual deste documento.
- Não citar ChromaDB como banco vetorial atual deste documento.
- Não tratar a stack do portfólio/Autobot como se fosse toda a stack técnica de Gabriel.
